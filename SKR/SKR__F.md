# 🛡️ СКОРО // Функциональное Ядро СКОРО (SKR__F)

Настоящий документ является полной спецификацией декларативного автомата управления, блоков вычислительной логики и синтаксиса внешних вызовов архитектуры **СКОРО**.

---

## ☢️ Декларативная Триада Группы `Алгоритмы`

Управление вычислительным процессом в СКОРО жестко разделено на три изолированные фазы автомата состояний. Это исключает динамические аллокации памяти, фрагментацию кучи и сбои синхронизации в критических комплексах (**СТРАЖ**, **ИСКРА**, **КЛЮЧ**).

### 0. `ops` // Опись Объектов Алгоритма
Глобальный декларативный сектор, определяющий всю структуру данных страницы.
*   **Назначение:** Описание структур, типов данных и выделение статической памяти под глобальные параметры числодробилки.
*   **Ограничение:** Запрещено размещать исполняемый код, математические операции и вызовы функций.

### 1. `dpsk` // Допуск Окна
Выполняется **до** Пуска Окна.
*   **Назначение:** Настрой Состояния Окна, настрой параметров безопасности, корректиров описанных ранее структур, инициализация регистров виртуальной машины `SKORO_RGST` и привязка оборудования.
*   **Ограничение:** Запрещено использовать Функциональные Прерывания (*ФП*) управления Объектами Отображения.

### 2. `psk` // Вычислительный Конвейер (Числодробилка)
Выполняется **после** Пуска Окна.
*   **Назначение:** Управление Окном Отображения: выполнение вычислительных операций и изменение параметров Объектов Отображения.
*   **Ограничение:** Запрещено использование Функциональных Прерываний управления Параметрами Безопасности.

### 3. `otpsk` // Отпуск Окна
Выполняется **перед** Опуском Окна.
*   **Назначение:** Сброс значений параметров и структур. Безопасный перевод аппаратных узлов в спящий режим, сохранение контрольных сумм параметров, фиксация логов.
*   **Ограничение:** Запрещено использование Функциональных Прерываний управления Параметрами Безопасности.

---

## 📋 Спецификация Внешних Вызовов (`@вызов_внешний`)

Поскольку ядро `SKORO_RGST` полностью изолировано от ОС и Vulkan SC, взаимодействие с Отображением происходит исключительно посредством вызова Функциональных Прерываний языка СКОРО. 

Каждое Функциональное Прерывание жестко регламентировано и идентифицируется парой параметров:

*   **`FP_KN`** — Кодовое Наименование Функционального Прерывания (текстовый маркер для линковщика).
*   **`FP_K_`** — Код Функционального Прерывания (шестнадцатеричный HEX-идентификатор для процессора ВМ).
*   **`ARG_N`** — Аргументы от 1 до 7 (в зависимости от структуры описи ФП), где `N` - номер Аргумента Функционального Прерывания.

### 🎛️ Синтаксическая Карта Вызова

Функциональные прерывания имеют ограниченное количество описей:

```text
FP_KN_0(FP_K)
FP_KN_1(FP_K, AGR_1)
FP_KN_2(FP_K_, ARG_1, ARG_2)
FP_KN_3(FP_K_, ARG_1, ARG_2, ARG_3)
FP_KN_4(FP_K_, ARG_1, ARG_2, ARG_3, ARG_4)
FP_KN_5(FP_K_, ARG_1, ARG_2, ARG_3, ARG_4, ARG_5)
FP_KN_6(FP_K_, ARG_1, ARG_2, ARG_3, ARG_4, ARG_5, ARG_6)
FP_KN_7(FP_K_, ARG_1, ARG_2, ARG_3, ARG_4, ARG_5, ARG_6, ARG_7)
```

---

### ⚙️ Механизм Транспиляции:
Транспилятор СКОРО не генерирует прямые ассемблерные переходы в сторонние библиотеки. Он регистрирует безопасные вызовы функций для компилятора `gcc`:
1.  **Изоляция вызова:** Контроль функций обеспечивается этапе сбора.
2.  **Передача по указателю:** Обращения к Объектам Отображения транспилируется в вычислительные операции.

## 📝 Комплексный Пример Реализации

Пример декларативной описи `@algrtm` (с использованием пятисимвольных HEX-типов из спецификации `SKR__T`):

```text
@algrtm {
	ops {
		// Пример Объявления Структуры
		struct s_okn_prm {
			t_n20	shrn;	// Ширина Окна
			t_n20	vst;	// Высота Окна
			t_d40	cvt;	// Цвет Окна
		} = {
			.shrn = 0x1,
			.vst = 0x0,
			.cvt = 0.0,
		};
	}

	dpsk {
		shrn	= 0x00000780;
		vst		= 0x00000438;
	}

	psk {
		cvt = 0.7
	}

	otpsk {
		vst = shrn = 0x0;
		cvt = 1.0
	}
}
```
