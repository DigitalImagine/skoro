# 📊 Спецификация Системы Типов Вычислительного Ядра СКОРО

Настоящий документ описывает архитектуру низкоуровневых типов данных, применяемых в изолированном вычислительном ядре (числодробилке) системы **СКОРО**.

---

## ⚡ Принципы Проектирования

Система типов разработана для применения в предсказуемых атомарных вычислических комплексах (**СТРАЖ**, **ИСКРА**, **КЛЮЧ**):

*   **Маркер `t_`:** Все типы данных без исключения начинаются с единого системного префикса.
*   **HEX-разрядность:** Размер типа в битах задается в шестнадцатеричной (16-ричной) системе счисления.
*   **Строгая моноширинность:** Идентификаторы всех базовых типов занимают ровно 5 символов.

---

## 📊 Моноширинная Система HEX-Типов

Идентификаторы всех базовых типов занимают ровно **5 символов**, обеспечивая табличную верстку исходного кода. Разрядность задается в шестнадцатеричной (16-ричной) системе счисления.

*   `08` (HEX) = 8 битов (1 байтов)
*   `10` (HEX) = 16 битов (2 байта)
*   `20` (HEX) = 32 бита (4 байта)
*   `40` (HEX) = 64 бита (8 байтов)
*   `80` (HEX) = 128 битов (16 байтов)

---

## 🗺️ Карта Базовых Типов

| Символ | Природа типа | Примеры (5 символов) | Описание типа |
| :--- | :--- | :--- | :--- |
| **`t_s`** | Знаковое целое | `t_s08`, `t_s10`, `t_s20`, `t_s40`, `t_s80` | `знаковое` |
| **`t_u`** | Беззнаковое целое | `t_u08`, `t_u10`, `t_u20`, `t_u40`, `t_u40` | `беззнаковое` |
| **`t_f`** | Дробное (IEEE 754)| `t_f20`, `t_f40` | `дробное`, `дробное (расширенное)` |


### 🟢 Беззнаковые Целые Числа (`t_u**`)
*   `t_u08` — 8-битное целое без знака (0x08).
*   `t_u10` — 16-битное целое без знака (0x10).
*   `t_u20` — 32-битное целое без знака (0x20).
*   `t_u40` — 64-битное целое без знака (0x40).
*   `t_u80` — 128-битное целое без знака (0x80).

### 🟡 Знаковые Целые Числа (`t_s**`)
*   `t_s08` — 8-битное целое со знаком (0x08).
*   `t_s10` — 16-битное целое со знаком (0x10).
*   `t_s20` — 32-битное целое со знаком (0x20).
*   `t_s40` — 64-битное целое со знаком (0x40).
*   `t_s80` — 128-битное целое со знаком (0x80).

### 🔵 Вещественные Числа / Дробные (`t_f**`)
*   `t_f20` — 32-битное число с плавающей точкой (IEEE 754).
*   `t_f40` — 64-битное число с плавающей точкой двойной точности.

## 📐 Аппаратное Выравнивание и «Младший Ноль»

Язык СКОРО не навязывает скрытых правил упаковки или принудительного выравнивания полей. Структуры размещаются в памяти в соответствии с описанием и стандартом компиляции. 

При этом физический размер типа в байтах жестко привязан к его HEX-идентификатору и вычисляется по формуле:

$$ \text{Размер в байтах} = 2 \times \text{Старший разряд HEX-числа} $$

*(Где старший разряд — это HEX-число без младшего разряда, полученное сдвигом `>> 4`)*

### Применение формулы:
*   `t_u10` $\rightarrow$ убираем младший разряд $\rightarrow$ $2 \times 1 = \mathbf{2}$ **байта** (16 битов)
*   `t_u20` $\rightarrow$ убираем младший разряд $\rightarrow$ $2 \times 2 = \mathbf{4}$ **байта** (32 бита)
*   `t_u40` $\rightarrow$ убираем младший разряд $\rightarrow$ $2 \times 4 = \mathbf{8}$ **байтов** (64 бита)
*   `t_u80` $\rightarrow$ убираем младший разряд $\rightarrow$ $2 \times 8 = \mathbf{16}$ **байтов** (128 битов)


1.  **Физический Младший Ноль:** Если инженер сам спроектировал структуру так, что её общий размер кратен 16 байтам (0x10), то при размещении массива таких структур в памяти, адрес начала каждого элемента будет оканчиваться на шестнадцатеричный `0` (в бинарном виде — младшие 4 бита равны `0000`).
2.  **Регистровая Оптимизация ВМ:** Зная, что структуры выровнены автором по границе 16 байтов, виртуальная машина `SKORO_RGST` может использовать неиспользуемые младшие 4 бита указателей в регистрах под системные флаги и маски безопасности. Перед отправкой адреса в шину памяти процессор делает аппаратный сдвиг (`<< 4`).

## 📜 Системный Заголовочный Файл (`SKORO__T.h`)

Интеграция с компилятором `gcc` реализуется через файл определений без привлечения тяжелых парсеров:

```c
#ifndef BLTK_SKORO__T___OPS
	#define BLTK_SKORO__T___OPS

	/*
		Размерность Регистров Указывается в битах в Шестнадцатеричной системе счисления
	*/

	// --- БЕЗЗНАКОВЫЕ ЦЕЛЫЕ (t_u) ---
	typedef uint08_t	t_u08;
	typedef uint16_t	t_u10;
	typedef uint32_t	t_u20;
	typedef uint64_t	t_u40;
	#ifdef __SIZEOF_INT128__
		typedef unsigned __int128 t_u80;
	#endif

	// --- ЗНАКОВЫЕ ЦЕЛЫЕ (t_s) ---
	typedef sint08_t	t_s08;
	typedef sint16_t	t_s10;
	typedef sint32_t	t_s20;
	typedef sint64_t	t_s40;
	#ifdef __SIZEOF_INT128__
		typedef __sint128	t_s80;
	#endif

	// --- ДРОБНЫЕ ТИПЫ (t_f) ---
	typedef float     t_f20;
	typedef double    t_f40;

#endif
```

## 🛠️ Пример Использования в Описях Структур

Благодаря пятисимвольному HEX-стандарту, блоки параметров в коде выстраиваются в безупречно ровные моноширинные таблицы:

```text
struct s_skoro__prm__1 {
    t_u08 статус
    t_u20 счетчик
    t_s10 дельта_x
    t_f40 таймстамп
}
```
